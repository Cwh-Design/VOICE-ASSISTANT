## 1. 产品概述

### 1.1 产品定位

一个面向中文用户的网页端 AI 语音对话助手，体验对标「中文版 Siri」。用户输入问题后，AI 经过"思考"再"回答"，回答内容以**打字机效果**逐字呈现，同时通过 **TTS 朗读出声**，并伴随以声波纹为核心主视觉的动态反馈。

### 1.2 核心价值

- **沉浸式视觉**：以 Strands 声波纹作为 AI 的"人格化"形象，状态切换有生命力。
- **声字同步**：打字机文字浮现速度与 TTS 语音朗读速度相匹配，形成"边说边出字"的自然体验。
- **极简交互**：一屏完成提问 → 思考 → 语音回答的完整闭环。

### 1.3 目标用户

希望快速获得 AI 语音回答的中文用户，适用于桌面端浏览器场景（移动端适配为次要目标）。

## 2. 页面整体框架

页面自上而下分为三层结构，垂直居中布局，背景为全屏 Aurora 动效。

```
┌─────────────────────────────────────────────┐
│              Aurora 全屏背景动效              │
│                                             │
│         ┌─────────────────────────┐         │
│         │   Strands 声波纹主视觉    │         │  ← 不限高度，偏高
│         │   （AI 人格化形象）       │         │
│         └─────────────────────────┘         │
│                                             │
│            轮询问题（SplitText 超大字号）       │  ← 纯白、超大字
│                                             │
│         ┌───────────────────────────┐       │
│         │  BorderGlow 输入框（偏高）  │       │  ← 圆角矩形
│         │                  [Ask 按钮] │       │  ← 白底黑字
│         └───────────────────────────┘       │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.1 布局规则

| 区域 | 内容 | 备注 |
| --- | --- | --- |
| 主视觉区 | Strands 声波纹 | 高度偏高，**不限制高度**，作为页面视觉中心 |
| 轮询区 | 多个预设问题轮播（SplitText） | 超大字号、纯白 |

## 3. 核心功能需求

### 3.1 AI 对话核心

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F-01 | 文本输入提问 | 用户在输入框输入中文问题，点击 Ask 或回车提交 |
| F-02 | 接入火山方舟大模型 | 使用火山方舟（Volcano Ark）API Key 调用大模型生成回答 |
| F-03 | 思考态展示 | 提交后进入"思考"状态，主视觉与文字区切换为 thinking 表现 |
| F-04 | 回答态展示 | 模型返回后进入"回答"状态，文字区切换为 answering 表现 |
| F-05 | 打字机文字输出 | 回答文字逐字浮现（ShinyText 效果呈现） |
| F-06 | TTS 语音朗读 | 回答内容同步用 TTS 朗读出声 |
| F-07 | 声字同步 | 打字机出字速度与 TTS 朗读速度匹配 |
| F-08 | 轮询问题展示 | 空闲态下预设问题以 SplitText 轮播展示，引导用户提问 |

### 3.2 状态机

系统共有三个核心状态，状态间通过**透明度渐隐渐现**过渡，禁止硬切换。

```
        提交问题                      TTS+文字播放完成
┌──────────────┐  ──────────►  ┌──────────────┐  ──────────►  ┌──────────────┐
│   idle 待命   │              │ thinking 思考  │              │ answering 回答 │
│ saturation=1  │              │ glass size=0.4 │              │  打字机+TTS    │
│ 轮询问题展示   │  ◄──────────  │  文字区:thinking│  ◄──────────  │  文字区:answering│
└──────────────┘    回到待命     └──────────────┘   模型返回+TTS就绪 └──────────────┘
```

| 状态 | Strands 参数 | 文字区内容 | 其他 |
| --- | --- | --- | --- |
| `idle` 待命 | `saturation = 1` | 轮询问题（SplitText） | 输入框可输入 |
| `thinking` 思考 | `glass size = 0.4` | 显示「thinking」（ShinyText） | 禁用输入，等待模型 |
| `answering` 回答 | 恢复正常参数 | 显示「answering」+ 打字机输出回答正文 | TTS 同步朗读 |

> **状态过渡要求**：所有 Strands 状态切换必须使用**透明度渐隐渐现（opacity fade）**方式过渡，不允许参数硬切换。实现思路：可叠加两个 Strands 实例/图层，新状态图层 opacity 0→1 渐入、旧状态图层 1→0 渐出，形成交叉溶解（crossfade）。

## 4. 技术架构

### 4.1 技术栈

| 层 | 选型 | 说明 |
| --- | --- | --- |
| 框架 | React + TypeScript | 单页应用 |
| 构建 | Vite | — |
| 样式 | Tailwind CSS | 配合 react-bits 组件 |
| 组件库 | shadcn + react-bits | 见第 5 节组件清单 |
| 字体 | HarmonyOS Sans（鸿蒙字体） | 全站统一 |
| 大模型 | 火山方舟（Volcano Ark） | 经后端代理，Key 不出服务器 |
| 语音合成 | 百度智能云·流式文本在线合成 | WebSocket，spd=15，流式播放 |

### 4.2 后端代理与 API 集成

所有第三方密钥**统一放后端**，前端只跟自己的后端通信，密钥永不暴露到浏览器。

- **后端代理服务**：自建一个小服务，代理两类请求——
  - 火山方舟对话 API（前端 → 后端 → 火山方舟）
  - 百度流式 TTS 的 WebSocket（前端 WS ↔ 后端 WS ↔ 百度 WS，透传音频帧）
- **火山方舟对话**：后端持有 Ark API Key，接收前端问题，调用大模型返回回答文本。
- **百度流式 TTS**（`wss://aip.baidubce.com/ws/2.0/speech/publiccloudspeech/v1/tts`）：
  - 后端用百度 API Key/Secret 换取 `access_token` 并建立 WebSocket。
  - 关键参数：`per`（发音人，待定）、`spd=15`（最快档）、`aue=3`（mp3）。
  - 文本按句发送（单次 ≤1000 字），服务端按句返回音频二进制，前端边收边播。
- **密钥管理**：所有 Key/Secret/Token 仅存于后端环境变量，**禁止**写入前端代码或出现在浏览器请求中。

### 4.3 声字同步逻辑（关键技术点）

采用**流式播放 + 按句同步**，兼顾"开口快"与"声字对齐"：

1. thinking 期间拿到完整回答文本，按标点（，。？！；等）切分成句子。
2. 经后端代理建立百度流式 TTS 连接（`spd=15`），按句发送 `text` 帧。
3. 百度按句返回音频二进制；**第一句音频一到就开始播放**（边合成边播，开口快）。
4. 每句音频到达后取其时长 `D_s`，将该句文字按 `D_s / 句字数` 的节奏逐字 reveal，与该句音频播放对齐。
5. 句与句顺序串联：播第 1 句 + 出第 1 句字，同时第 2 句在后台合成，依次接续。
6. **权威时钟**：以音频实际播放进度为准校准文字 reveal，避免漂移；若某句时长未取到，按 `spd=15` 估算字符速率兜底。

> 语速固定用百度 `spd=15`（最快档，非浏览器 3 倍速播放）；流式播放，第一句几乎立刻出声。

## 7. 交互流程（用户旅程）

### 7.1 主流程

1. **进入页面**：Aurora 背景启动，Strands 以 `saturation=1` 待命，轮询问题以 SplitText 超大字号纯白轮播。
2. **输入提问**：用户在 BorderGlow 输入框输入问题。
3. **提交**：点击 Ask（或回车）。
4. **进入 thinking**：
   - Strands 通过 opacity 渐隐渐现切换到 `glass size=0.4`。
   - 文字区切换为 ShinyText 显示「thinking」。
   - 输入框禁用。
   - 同时调用火山方舟对话 API。
5. **进入 answering**：模型返回 + TTS 音频就绪后：
   - Strands 通过 opacity 渐变恢复正常参数。
   - 文字区切换为 ShinyText 显示「answering」，并开始打字机逐字输出回答正文。
   - TTS 以 `speed=3` 播放，打字机出字节奏与音频对齐。
6. **回到 idle**：TTS 播放完毕、打字机出字完成 → 回到待命，轮询问题恢复轮播，输入框重新可用。

### 7.2 状态参数对照表

| 时刻 | Strands | 文字区 | 输入框 | 音频 |
| --- | --- | --- | --- | --- |
| idle | saturation=1 | 轮询问题(SplitText) | 可用 | — |
| thinking | glass size=0.4 | 「thinking」(ShinyText) | 禁用 | — |
| answering | 恢复正常 | 「answering」+ 打字机正文(ShinyText) | 禁用 | 百度流式 spd=15 |

## 8. 非功能性需求

| 类别 | 要求 |
| --- | --- |
| 性能 | Strands / Aurora 动效需保持 60fps，避免主线程阻塞；TTS 音频采用流式优先 |

| 可访问性 | TTS 朗读提供静音开关（可选增强项） |
| 国际化 | 首版仅中文 |

## 10. 验收标准

1. ✅ 页面背景为 Aurora 全屏动效，无白屏/卡顿。
2. ✅ 主视觉为 Strands 声波纹，高度偏高且**未限制高度**。
3. ✅ Strands 在 idle 态 `saturation=1`，thinking 态 `glass size=0.4`。
4. ✅ Strands 状态切换为**透明度渐隐渐现**，无明显硬切。
5. ✅ 轮询问题以 SplitText 展示，**超大字号、纯白**。
6. ✅ 输入框为 BorderGlow 圆角矩形，高度偏高；Ask 按钮**白底黑字**。
7. ✅ 全站使用**鸿蒙字体（HarmonyOS Sans）**。
8. ✅ 接入火山方舟 API Key，输入问题后能思考并返回回答。
9. ✅ 思考态文字区显示「thinking」，回答态显示「answering」，均用 ShinyText。
10. ✅ 回答以**打字机效果**逐字出现，并经**百度流式 TTS 朗读**，`spd=15`，边合成边播、按句声字同步。
11. ✅ 打字机出字速度与 TTS 朗读速度**视觉同步**，无显著超前/滞后。
